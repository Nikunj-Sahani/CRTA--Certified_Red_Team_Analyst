### Network Pivoting
Network Pivoting means using a **compromised system to access other systems** inside the internal network.

- 👉 If an attacker **hacks one machine**, they can use it as a bridge to **reach other machines** that are not directly accessible.

### 📌 Example

> - 1️⃣ Attacker hacks Machine A
> - 2️⃣ Machine A is connected to the internal network
> - 3️⃣ Attacker uses Machine A to attack Machine B and C

### 🎯 Purpose

  > - Access hidden/internal systems
  > - Move deeper inside the network
  > - Find more vulnerabilities
