---
- [ ] **Cyber Kill Chain is focussed on obtaining the information related to our target machine.**
  
   - [X] Using Tools we can collect all the important information related to our target.

### Steps in Cyber Kill Chain
6 basic steps are included here.

  > - **Reconnaissance –** Collecting information about the target

  > - **Scanning & Enumeration –** Identifying open ports, services, and resources

  > - **Gaining Access –** Exploiting vulnerabilities to enter the system

  > - **Post Exploitation –** Expanding access and gathering more data

  > - **Maintaining Persistence –** Keeping long-term access to the system

  > - **Removing Footprints –** Hiding or deleting attack traces

---
## 👁️ Reconnaissance
Information-Gathering phase : *collecting data about a target to find weaknesses and plan the attack.*

### ❇️ Types of Reconnaissance

- **Passive Recon :** Collecting information without directly contacting the target.
     > - *Examples :* WHOIS, DNS records, public web pages, LinkedIn, job posts, leaked credentials, Google dorking.
  
- **Active Recon :** Directly probing the target’s systems or people, which can be detected.
     > - *Examples :* Examples: port scans, vulnerability scans, banner grabbing, brute-force attempts.

### ❇️ Some Tools for information Gathering
 - WHOIS Lookup (Use in Linux)
 - Recon-ng (Use in Linux)
 - Nmap
 - Arp-Scan
 - Netdiscover

## 👁️ Scanning & Enumeration

- **🏗️ Nmap - (Network Mapper)**
 One of the most popular and versatile port scanning tools.
  - [X]  It helps discover hosts, services, operating systems, and open ports.

 - **Example Command :- nmap IP address**

   > - **nmap -sn 192.168.1.0/24  ::-** Ping Scan
   
   > - **nmap -sn -PR 192.168.1.0/24  ::-** ARP Scan (LAN only)

   > - **nmap -sT 192.168.1.10  ::-** TCP Connect Scan

   > - **nmap -sU 192.168.1.10  ::-** UDP Connect Scan

   > - **nmap -sS 192.168.1.10  ::-** SYN Scan (Stealth Scan)
   
   > - **nmap -sF 192.168.1.10  ::-** FIN Scan

   > - **nmap -sN 192.168.1.10  ::-** NULL Scanś

   > - **nmap -sV 192.168.1.10  ::-** Service Version Detection

   > - **nmap -A 192.168.1.10  ::-** Aggressive Scan
   
   > - **nmap -p- 192.168.1.10  ::-** All Port Scan

   > - **nmap 192.168.1.10 -p0-1000 ::-** Port scan in limitation
   
   > - **nmap -O 192.168.1.10  ::-** OS Detection
   
   > - **nmap -sC 192.168.1.10  ::-** Check Scripts
   
   > - **nmap --script=vuln 192.168.1.10  ::-** Checkout CVE if got
   
   > - **nmap target IP -D 192.168.1.10 -p 21 ::-** Check network in Wireshark

## 👁️ Gaining Access
It is the phase where an attacker exploits identified vulnerabilities to obtain:
  
   - [ ] Remote shell access
   - [ ] User credentials
   - [ ] Admin/root privileges
   - [ ] Access to services or databases

This happens after reconnaissance and scanning.

---
## 👁️ Post-exploitation
 The phase where, after getting initial access (like a shell), the attacker or ethical hacker:

   - [ ] Expands privileges
   - [ ] Moves to other systems
   - [ ] Extracts sensitive data
   - [ ] Establishes persistence
   - [ ] Maps the internal network

⚠️ Always perform only in authorized lab environments (like Metasploitable, HTB, THM).

---
## 👁️ Maintaining Persistence
Persistence means creating a way to access a system again later without re-exploiting the original vulnerability.

- After exploitation, an attacker may:

 - [ ] Lose connection
 - [ ] System may reboot
 - [ ] Password may change
 - [ ] Vulnerability may get patched
 
In real attacks, this techniques allows long-term control.

   - [X] Adding a New user
   - [X] Adding SSH key access
   - [X] Backdoor web shell
   - [X] Startup scripts

---
## 👁️ Removing Footprints
This phase refers to attempts by an attacker to:

 - [ ] Delete logs
 - [ ] Clear command history
 - [ ] Remove created accounts
 - [ ] Erase malware files
 - [ ] Disable monitoring tools
